  3. Engine locking

    在 Next.js 的项目配置中，engine locking 是一个选项，它控制 Next.js 是否应该锁定使用的 Node.js 引擎版本。

    当启用 engine locking 选项时，Next.js 将在项目中使用的 Node.js 引擎版本上锁定，这可以确保在构建和运行应用程序时使用相同的 Node.js 版本。这可以帮助避免由于不同版本的 Node.js 引擎之间的差异导致的问题，从而使应用程序更加稳定和可靠。

  6. lint-staged

    有时想上传代码，但是并不是所有的文件都符合 lint。如果不装 lint-staged，所有的文件就都要满足 lint 的要求。如果有 lint-staged，只有 staged 文件才需要满足 lint 要求。即，commit 中的 changes 外，又多了一个 staged changes。

    2. git hooks (husky)

      Git hook（husky）是一个 Git 的客户端钩子工具，可以让开发者在 Git 事件发生前或者后，执行一些自定义脚本。通过使用Git hook，可以自动化地完成代码规范检查、单元测试、代码打包、自动部署等操作，提高开发效率，减少错误。

      常见的 Git hook 包括 pre-commit、post-commit、pre-push 等。其中，pre-commit 是在代码提交前执行，可以用来检查代码风格、运行单元测试等操作；post-commit 是在代码提交后执行，可以用来更新文档、发送邮件等操作；pre-push 是在代码推送前执行，可以用来检查代码质量、运行端到端测试等操作。
